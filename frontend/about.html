1. What is Aura?

Aura is a production-grade Sentiment Analysis web application designed as a comprehensive final-year / capstone project. Unlike simple polarity checkers, Aura performs 8 distinct NLP analyses on every piece of text, delivering a multi-dimensional emotional intelligence report.

The platform supports single-text analysis with real-time typing detection, bulk CSV/TXT file uploads (up to 50 texts per batch), a personal analytics dashboard with Chart.js visualizations, a full admin panel for platform-wide management, and Gemini AI–powered explanations and tone rewrites.

When you type or paste text into the Aura Analyzer, the following pipeline executes in milliseconds:

User types text in the Analyzer
Auto-detect on pause or manual "Analyze" click
Step 2
Frontend sends POST /api/analyze
Firebase Bearer token attached for authentication

NLP Service processes all 8 analyses
TextBlob → NRCLex → spaCy → Sarcasm → Toxicity → Sentences → Keywords

Step 5
Rich results rendered in the frontend
Sentiment card, emotions, entities, aspects, sarcasm, toxicity, sentences, keyword map

3. The NLP Pipeline — 8 Analysis Features

All features are orchestrated by a single unified analysis function that runs every processor in sequence and returns a comprehensive result object.

Core

① Sentiment Polarity

Uses TextBlob to compute a numerical polarity score (−1.0 to +1.0). Text with polarity > 0.1 is classified Positive, below −0.1 is Negative, and everything else is Neutral.

Confidence = |polarity| × 1.5, capped at 1.0

NRC

② Emotion Extraction

Powered by NRCLex (NRC Emotion Lexicon), this maps every word against 8 emotion categories: joy, trust, fear, surprise, sadness, disgust, anger, and anticipation. Top 3 emotions are returned.

Filters out generic "positive"/"negative" tags for cleaner results

spaCy

③ Named Entity Recognition

The en_core_web_sm ML model identifies real-world entities: people (PERSON), organizations (ORG), locations (GPE), dates (DATE), monetary values (MONEY), and more. Up to 5 unique entities returned.

Only runs when language is detected as English

spaCy

④ Aspect-Based Sentiment

Uses spaCy's dependency parsing + noun chunks to isolate individual topics (aspects) in a sentence, then calculates each aspect's polarity using a ±3 token context window via TextBlob.

e.g., "The camera is great but battery is terrible" → camera: Positive, battery: Negative

Heuristic

⑤ Sarcasm Detection

A multi-signal heuristic scorer analyzing: 20+ sarcasm phrases ("oh great", "yeah right"), positive-polarity with negative emotions contradiction, excessive punctuation (!!!), ALL CAPS words, and contradictory emoji (🙄😒).

Score 0–1 → Sincere (<0.25), Possibly Sarcastic (0.25–0.45), Likely Sarcastic (≥0.45)

Filter

⑥ Toxicity / Hate Speech

Two-tier word-list classification: TOXIC_WORDS (high-severity: violence-related terms) and OFFENSIVE_WORDS (medium-severity: insults and slurs). Returns label (Safe/Offensive/Toxic), a severity score, and matched words.

Toxic base score: 0.6 + 0.1 per match; Offensive base: 0.3 + 0.08 per match

TextBlob

⑦ Sentence-Level Breakdown

Splits the input into individual sentences using TextBlob's sentence tokenizer, then computes independent polarity per sentence. This reveals mixed-sentiment texts where overall polarity might be misleadingly neutral.

Each sentence tagged Positive/Negative/Neutral with its exact polarity score

Hybrid

⑧ Keyword Sentiment Map

A hybrid of TextBlob per-word polarity + NRCLex emotion tagging. First scores every non-stopword via TextBlob, then augments unscored words using NRC's affect dictionary (positive/negative tags). Top 15 keywords by absolute polarity are returned.

Color-coded: green (positive), red (negative), gray (neutral)

4. Gemini AI Intelligence

Beyond local NLP, Aura integrates Google Gemini 2.5 Flash for two advanced AI-powered features accessible via dedicated service and API layers:

✨ AI Explanation

After analysis, the user can click "Explain with AI". Gemini receives the full analysis context (sentiment, confidence, emotions, entities) and generates a 2–3 sentence human-readable narrative explaining why the text carries its sentiment.

Endpoint: POST /api/ai/explain

🔄 Tone Rewriter

Gemini rewrites the user's text into 3 different tones: Positive (optimistic, upbeat), Neutral (balanced, factual), and Formal (corporate, professional). Results are returned as structured JSON.

Endpoint: POST /api/ai/rewrite

5. Backend Architecture

The backend is built with Python 3 + FastAPI using a clean modular architecture. Routers are auto-prefixed under /api and the frontend is served statically from the same server.

Application Entry Point

Initializes FastAPI with lifespan events, registers all 4 API routers, mounts static CSS/JS directories, and defines routes for serving all frontend HTML pages. Runs via Uvicorn with hot-reload.

Core Analysis API

POST /api/analyze: Single text analysis → runs full NLP pipeline → saves to Firestore → returns result. POST /api/analyze/bulk: Accepts CSV/TXT upload → processes up to 50 texts in a batch → commits all to Firestore via batch writes → returns aggregated summary.

User Data API

GET /api/recent: Fetches the user's 15 most recent analyses sorted by timestamp. GET /api/user/stats: Returns total counts + sentiment trend data for Chart.js. GET /api/user/analytics: Deep analytics with sentiment history + word frequency map for the word cloud.

Admin Panel API

Platform-wide statistics with pagination & filtering, daily sentiment trend data, global word frequency analysis with sentiment tagging, query deletion, user management (list/delete), and full CSV export of all platform data.

Gemini AI API

Two endpoints for AI-powered features: POST /api/ai/explain (narrative explanation) and POST /api/ai/rewrite (tone rewriting). Both require authentication and delegate to the Gemini service layer.

6. Frontend Modules

🔮 The Analyzer Engine

The core interface with tabbed input (text mode with auto-analyze on pause, file upload with drag-and-drop). Renders all 8 NLP results in dedicated UI panels: sentiment card with emoji, emotion tags, entity badges, aspect pills, sarcasm meter, toxicity shield, sentence-by-sentence breakdown with color coding, and a keyword sentiment cloud. Includes "Explain with AI" and "Rewrite Tone" buttons.

📊 Personal Analytics Dashboard

Three-tab dashboard: Overview (KPI stat cards with animated counters, sentiment distribution pie chart, word cloud), History (scrollable list of past analyses with re-analyze capability), and AI Profile (Gemini-generated personality insights based on your analysis history). Chart.js powers all visualizations.

🛡️ Admin Control Panel

Platform-wide management interface with real-time stats, paginated query browser with sentiment filtering, daily sentiment trend charts, word frequency word cloud, user management table (with delete capability), and one-click CSV data export.

🔐 Authentication Portal

Firebase Authentication with email/password and Google Sign-In. Handles login, registration, and session management with ID token extraction for API authorization.

🔔 Notification System

Custom toast notification system with animated slide-in/out, multiple severity levels (success, error, warning, info), auto-dismiss timers, and stacking support for multiple simultaneous notifications.

7. Authentication & Security

Frontend Auth (Firebase Client SDK)

Firebase Authentication handles user registration and login via email/password and Google OAuth. Upon successful login, the Firebase Client SDK generates an ID token, which is stored in session and attached as a Bearer token to every API request's Authorization header.

Backend Verification (Firebase Admin SDK)

The security module provides a FastAPI dependency that extracts the Bearer token from every incoming request, verifies it against Firebase using the Admin SDK, and returns the decoded user payload (UID, email) to route handlers — ensuring only authenticated users can access protected APIs.

8. Database & Cloud Infrastructure

Aura uses Google Cloud Firestore — a serverless, real-time NoSQL document database chosen for its scalability, automatic indexing, and seamless integration with Firebase Auth.

Collection: queries

Each document stores the complete analysis result with these fields:

text sentiment confidence language emotions (JSON) entities (JSON) aspects (JSON) uid timestamp (SERVER_TIMESTAMP)

Initialization handled gracefully in config.py — if Firebase credentials are missing, the app falls back to local-only mode with no data persistence.
